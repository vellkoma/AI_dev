PART 01

世界のAI開発 — 種類別一覧

2026年7月時点で世界的に開発が進んでいるAI分野を11種類に整理。今年の二大テーマは「AIエージェント」と「フィジカルAI」で、生成AIは“試す年”から“業務に組み込む年”へ移行しています。

基盤技術

大規模言語モデル（LLM）

各社フラッグシップの競争が継続。自律的なタスク完遂能力や100万トークン級の長文処理が競争軸。汎用モデルから業務特化モデルへの選別も進行。

主要: OpenAI GPT-5系 / Google Gemini 3.1 / Anthropic Claude
2026年 最重要

AIエージェント（自律型AI）

曖昧な目標から一連のワークフローを自律完遂。「ツール」から「同僚」へ。Gartnerは2028年までにB2B購買の90%がエージェント仲介と予測。ただし本格導入企業はまだ15%未満。

主要: Salesforce Agentforce / 各社エージェント基盤 / MCP
投資が急増

フィジカルAI / ロボティクス

VLA（Vision-Language-Action）モデルの進化で「ロボティクスのChatGPTモーメント」が到来。2025年のVC投資は60億ドル超。ヒューマノイドや世界モデルの開発競争が加速。

主要: NVIDIA（Cosmos / Isaac / GR00T）/ Figure AI / Boston Dynamics
標準機能化

マルチモーダルAI

テキスト・画像・音声・動画を統合処理。最新フラッグシップLLMはほぼ標準対応。請求書OCR、画像つきレポート生成、現場写真の問題点抽出などに活用。

主要: GPT-5 / Gemini / Claude 等の最新モデル群
再編期

画像・動画・3D生成AI

Adobe Fireflyなどクリエイティブ分野の開発は継続する一方、OpenAIはSoraを終了しエージェント開発へリソース集中。3D生成はEC・不動産VR・ゲームで急成長中。

主要: Adobe Firefly / Midjourney / 各社動画モデル
開発の民主化

コーディング支援AI

定型コーディングをAIに任せるスタイルが定着し開発速度は平均40%以上向上。自然言語だけでアプリを構築する「開発の民主化」が進行。

主要: Claude Code / GitHub Copilot / Cursor
検索の置換

AI検索 / RAG

従来型検索エンジンを置き換える競争が継続。日本発のFeloが多言語対応を武器に全世界300万ユーザー超へ急成長。企業内では社内文書検索（RAG）需要が拡大。

主要: Perplexity / Genspark / Felo
機密データ対応

ローカルAI / SLM

スマホ・PCで動くオンデバイスAIが普及期に。医療・金融などクラウドに送れないデータの処理で決定的な利点。クラウドとのハイブリッド運用が主流になる見通し。

主要: 各社SLM / オンデバイス推論チップ
企業導入の中心

業界特化型AI（垂直統合型）

汎用チャットボットのブームを脱し、製造の設計自動化・技術伝承、金融の契約書照合・コンプラチェックなど、業務ドメイン特化型AIが普及の中心に。

領域: 製造 / 金融・保険 / 建設 / 物流 / 医療
推論シフト

AIインフラ（推論・半導体）

2026年はAI計算リソースの約3分の2が推論（inference）に使われる予測。「作るフェーズ」から「使い倒すフェーズ」へ移行し、データセンター・半導体投資が世界的に拡大。

主要: NVIDIA / クラウド大手 / 推論最適化基盤
研究の最前線

基盤モデル開発（研究開発）

LLMそのものを作る領域。世界モデル・長期記憶・推論能力の研究が中心で、参入は研究機関と一部大手に限られる。転職市場では極少数の博士級ポジション。

主要: OpenAI / Google DeepMind / Anthropic / Meta
PART 02

学習優先順位 — 転職価値 × 習得しやすさ

「AIモデルをゼロから作る側」ではなく「AIを使ってシステムを作る側（AI活用エンジニア）」から入るのが、未経験からの転職では最も現実的で市場価値も高い戦略です。

順位 = 未経験からの学習投資に対するリターン 5段階評価（ゲージ本数）
1RANK

LLM活用開発（API連携）

OpenAI / Claude / Gemini のAPIを使ったアプリ開発。全ての土台。
市場価値5/5
難易度2/5
求人数が最も多く、Python基礎があれば数ヶ月で形になる。最初の一歩はここ一択。
2RANK

「自社文書に答えるAI」の構築技術。ベクトルDB・埋め込み検索を含む。
市場価値5/5
難易度3/5
企業導入案件の中心。①と合わせて習得すると即戦力として評価されやすい。
3RANK

AIエージェント開発

MCP・ツール連携・マルチエージェント設計。2026年最大のトレンド。
市場価値5/5
難易度3/5
経験者がまだ少なく、ポートフォリオで最も差別化しやすい領域。
4RANK

AIコーディングツール活用

Claude Code / Cursor / Copilot。学ぶ対象であり学習を加速する道具。
市場価値4/5
難易度1/5
今や「使えて当たり前」になりつつある。学習初日から併用すべき。
5RANK

プロンプトエンジニアリング / 評価（Evals）

出力品質の設計と定量評価。①〜③の品質を左右する基礎技術。
市場価値3/5
難易度2/5
単体では武器になりにくいが、評価（Evals）まで扱えると価値が上がる。
6RANK

業界特化型AI導入（ドメイン × AI）

前職の業務知識（製造・金融・医療・物流など）とAIの掛け算。

機密データを扱う企業案件で需要増。インフラ・GPU知識が前提。

運用・監視・コスト最適化。クラウド経験が前提の高単価領域。

PART 03

学習ロードマップ（6〜12ヶ月想定）

転職の武器になるのは資格よりも「動くものをGitHubと記事で公開している」事実。各ステップでポートフォリオを1本ずつ積み上げます。

STEP 1 ── 1〜2ヶ月目

Python基礎 + LLM API入門

Pythonの基本文法を押さえたら、すぐにLLMのAPIを叩いてチャットアプリを作る。「AIに何ができて何ができないか」の肌感覚を掴む。AIコーディングツール（Claude Code / Cursor）は初日から併用して学習を加速させる。

成果物: 自作チャットアプリ 1本
STEP 2 ── 2〜4ヶ月目

RAGアプリを1本作る

「PDFをアップロードすると内容について答えてくれるAI」など。ベクトルDB（埋め込み検索）の仕組みを理解すると、企業案件で最も多いパターンに対応できるようになる。

成果物: 文書検索QAアプリ 1本
STEP 3 ── 4〜6ヶ月目

AIエージェントに発展させる

検索・ファイル操作・外部サービス連携など複数のツールをAIに使わせる構成へ。ここまで作れる人はまだ少なく、転職市場での明確な差別化ポイントになる。

成果物: ツール連携エージェント 1本
STEP 4 ── 全期間で並行

ポートフォリオ化・発信

GitHub公開＋技術記事（Zenn / Qiita / note）で開発過程を発信する。日本の求人ではG検定・E資格が評価されることもあるが、あくまで補助と位置づける。

成果物: 公開リポジトリ 3本 + 記事数本

最大の差別化戦略 — 「前職ドメイン × AI」

「AIが得意な未経験者」は増えていますが、「〇〇業界を知っていてAIも作れる人」は希少です。経理出身なら請求書処理AI、営業出身なら商談要約エージェント——前職の業務知識とAIの掛け算が、そのまま面接での最強の武器になります。ポートフォリオの題材は必ず自分のドメインから選んでください。
